# Production install (Ubuntu 22.04 + GPU + ViciDial)

End-to-end checklist for a real deployment. Plan ~90 minutes for a fresh
server, longer if downloading large model weights for the first time.

---

## 1. Provision the server

| Component | Minimum | Recommended |
|-----------|---------|-------------|
| OS        | Ubuntu 22.04 LTS server | Ubuntu 22.04 LTS server |
| CPU       | 8 cores | 16 cores |
| RAM       | 32 GB  | 64 GB |
| Disk      | 200 GB SSD | 500 GB NVMe |
| GPU       | 1× NVIDIA with 24 GB VRAM (RTX 4090, A5000, A100) | 2× same, or 1× A100 80 GB |
| Network   | 1 Gbps, static IP, DNS A-record | 10 Gbps, dedicated SIP IP |

**Why a GPU is non-negotiable:** vLLM runs the LLM, faster-whisper runs
STT, and F5-TTS synthesises voice — all need CUDA. Running them on CPU
is slow enough that calls drop out before the model finishes thinking.

**Cloud equivalents** (per-hour, cheapest first):
- Lambda Labs `gpu_1x_a10` (~$0.75/h, 24 GB) — fine for one campaign
- Runpod `RTX 4090` (~$0.35/h spot, $0.69/h on-demand)
- AWS `g6.2xlarge` (~$1/h, 24 GB L4)
- GCP `g2-standard-8` (~$1.20/h, 24 GB L4)

For sustained production, owning a tower with an RTX 4090 ($1500-2500
one-time) usually beats cloud after ~3 months.

---

## 2. Open the right network ports

| Port | Direction | What |
|------|-----------|------|
| 22   | inbound from your jump box only | SSH |
| 443  | inbound (public) | Panel HTTPS (nginx) |
| 5060 | inbound + outbound from ViciDial / SIP carrier | SIP signalling |
| 16384–32767 | inbound + outbound | RTP audio |

Lock 22 and 443 to your office IPs if you can. SIP/RTP needs to be open
to whatever IPs your ViciDial server and SIP carrier use.

---

## 3. ViciDial prereqs

You need a working ViciDial install reachable from this server. The
panel does not bundle ViciDial — get it from <https://vicidial.com>
(community edition is free) or use a managed install.

What you need to know about your ViciDial:
- Public web URL (e.g. `https://vici.acme.com/`)
- Asterisk Manager (AMI) host + port + user + password
- An admin web user that can read campaigns + ingroups
- One agent slot (web user + phone login + phone password) per AI agent
  you plan to deploy

The Session Manager logs into ViciDial as that agent slot — same as a
human agent would. If your ViciDial is on a private network, this server
must be able to reach both the web UI and the SIP/AMI ports.

## 4. Install

```bash
# As a sudo-capable user
sudo apt update && sudo apt install -y git
git clone https://github.com/steven-patrick18/aipanel.git /opt/aipanel
cd /opt/aipanel
sudo ./install.sh
```

What `install.sh` does (45-90 min depending on hardware + bandwidth):

1. Installs OS deps (Python 3.11, Node 20, Postgres 16, Redis, MinIO,
   nginx, NVIDIA drivers + CUDA toolkit)
2. Compiles PJSIP from source
3. Creates the `aipanel` system user + directory layout
4. Bootstraps Postgres (`aipanel` DB + `aipanel` user) and pgvector
5. Applies the SQL migrations
6. Creates per-service Python venvs and installs each service
7. Downloads model weights from HuggingFace into
   `/var/lib/aipanel/models/` (Qwen2.5-14B-AWQ, faster-whisper large-v3,
   F5-TTS, BAAI/bge-m3) — biggest single step, ~30 GB total
8. Installs Playwright + Chromium for the Session Manager
9. Builds the React frontend → `/var/www/aipanel/`
10. Generates a self-signed nginx cert (replace with real cert later)
11. Drops systemd units for every service, starts them
12. Drops a sudoers fragment that lets the panel UI's "Apply update"
    button run `update.sh`

Watch progress in another terminal:

```bash
tail -f /var/log/aipanel/install.log
```

## 5. Configure

Edit `/etc/aipanel/aipanel.conf` — defaults are sensible; the things
you'll likely change:

```toml
[panel]
public_url = "https://aipanel.acme.com"   # what the browser hits
cors_origins = ["https://aipanel.acme.com"]

[sip]
public_ip = "203.0.113.42"   # this server's public IP, for RTP NAT

[llm]
model = "Qwen/Qwen2.5-14B-Instruct-AWQ"   # swap if you have more VRAM
gpu_memory_utilization = 0.75              # raise on dedicated boxes

[stt]
model = "large-v3"
device = "cuda"
compute_type = "float16"
```

Secrets (passwords, JWT key) live in `/etc/aipanel/secrets.env` and were
auto-generated by `install.sh`. Don't commit them.

After edits:

```bash
sudo systemctl restart aipanel-web aipanel-llm aipanel-stt aipanel-tts
```

## 6. Replace the self-signed cert

`install.sh` generates a self-signed cert so the panel works on day one.
For real use:

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d aipanel.acme.com
```

(Or drop your existing cert at `/etc/aipanel/tls/cert.pem` + `key.pem`
and reload nginx.)

## 7. Create the first admin

```bash
sudo /opt/aipanel/scripts/bootstrap_admin.sh \
    --tenant=Acme \
    --email=ops@acme.com \
    --password='pick-something-strong-at-least-12-chars'
```

## 8. Sign in + bring up the platform

Open `https://aipanel.acme.com/login` and sign in.

Order matters because each step unlocks the next:

1. **System → Health** — green across the board?
2. **System → Cluster** — node showing as `ok` with all 5 services?
3. **ViciDial servers → Add server** — your ViciDial creds. Hit
   `Test connection` — should be green.
4. **Voices → Clone voice** — record / upload a 30-60s reference clip.
5. **Knowledge bases** (optional) — upload product PDFs / FAQs.
6. **Campaigns → New campaign** — pick a methodology.
7. **Agents → New agent** → fill **Persona** → **Training** tab → paste
   the script + upload real call recordings → **Promote to ready**.
8. **Deployments → New deployment** — pick agent + ViciDial server.
   Campaigns + ingroups dropdowns auto-populate from the dialler.
9. Open the deployment → **Start**. Session Manager logs into ViciDial
   and waits for inbound calls.

---

## 9. Verify with a test call

From the agent detail page → **Test call** → enter a real phone number
you have access to. ViciDial dials, the AI picks up when the leg
connects, you hear the agent's opening line. Hang up, then go to
**Calls** → click the most recent one to see the transcript + recording.

## 10. Updates

Once a new release is tagged on the GitHub repo:

**Option A — UI** (recommended):
**System → Updates → Apply update**. The panel runs `update.sh` for
you, streams the log, auto-rolls-back on failure.

**Option B — CLI**:
```bash
sudo /opt/aipanel/update.sh --yes
```

Either way: DB is backed up first, services restart in dependency order,
auto-rollback on any failure. To revert manually:
```bash
sudo /opt/aipanel/update.sh --rollback
```

---

## 11. Backups

`install.sh` enables a daily cron at 03:00 that runs
`/opt/aipanel/backup.sh`, dumping Postgres + config to
`/var/lib/aipanel/backups/`. Mirror this to S3/B2/etc:

```bash
# Add to /etc/cron.daily/aipanel-offsite-backup, for example
rclone sync /var/lib/aipanel/backups remote:aipanel-backups
```

To restore:
```bash
sudo /opt/aipanel/restore.sh --from=/path/to/backup.tar.gz
```

## 12. Operational basics

```bash
# Check everything's up
sudo /opt/aipanel/status.sh

# Tail logs across all services
sudo /opt/aipanel/logs.sh

# Restart one service
sudo systemctl restart aipanel-web

# Check active calls
curl -s http://127.0.0.1:9101/metrics | grep aipanel_worker_active_calls

# Disk + GPU
df -h /var/lib/aipanel
nvidia-smi
```

## Common first-day issues

- **`nvidia-smi` not found after install** — driver install needed a
  reboot. `sudo reboot`, then re-run `sudo ./install.sh` (idempotent).
- **vLLM OOM on first call** — `gpu_memory_utilization = 0.6` is the
  safe default. Lower it further if you share the GPU with other things.
- **Session Manager can't log into ViciDial** — the most common cause
  is firewall / DNS. From this box: `curl -v https://your-vici/`.
- **Calls connect but no audio** — RTP NAT. Set `[sip].public_ip` to
  this server's public IP, restart `aipanel-sip`.
- **"Apply update" button greyed out** — the sudoers fragment didn't
  install. `sudo visudo -c -f /etc/sudoers.d/aipanel-update` to check;
  re-run `sudo /opt/aipanel/install.sh` to reinstall it.

## What it costs to run

- Server: $50-200/month cloud, or one-time $1500-3000 on-prem
- Bandwidth: ~50 KB/min per concurrent call (Opus codec)
- Power on-prem: ~400W average with one GPU, ~$30/month at $0.10/kWh
- ViciDial trunking: whatever your SIP carrier charges per minute

No per-seat licensing, no per-call fees from this software — your only
recurring software cost is your SIP carrier.
